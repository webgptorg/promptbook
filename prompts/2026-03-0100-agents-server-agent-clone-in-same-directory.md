[✨🗄] Agent clone in same directory

-   In the [Agents Server](apps/agents-server) UI, users can create an agent based on an existing one
currently, when cloning an agent that is placed in some directory/folder, the clone ends up in root. This creates extra manual work to move the clone back to the same directory.
-   Implement cloning so that a newly created cloned agent is created in the same directory (folder) as the source agent by default.
-   UX requirements:
    -   The clone action should keep the existing behavior
-   Add the changes into the [changelog](changelog/_current-preversion.md)
