**This is how the prompt to agent runner should look like:**

```markdown
Answer 1 user question

-   Look at folder [user question](PATH_TO_FILE) and answer it
-   Start your answer with line containing "MESSAGE @Agent"
-   Only thing you should change in the repository is to add answer to the most recent message, nothing else
-   You should behave according to [your instructions](agent.book) which are explained in [the book manual](docs/book-language-manual.md)
-   The knowledge you are working with is in the `knowledge` folder, you can use it to find the answer to the question. You can also use it to find relevant information that can help you answer the question.
```

**This is how the typical chat in `messages/queued` looks like:**

```markdown
MESSAGE @User
How many events are in my calendar for this week?
```

**This is how the typical chat in `messages/finished` looks like:**

```markdown
MESSAGE @User
How many events are in my calendar for this week?

MESSAGE @Agent
There are 5 events in your calendar for this week.
```

-   Theese commands are cousins to the `ptbk coder` commands but they will be used for non-coding agents
-   Purpose of `ptbk agent` is to run complex agent behaviour emulated by the run of the coding agent. There will be other software _(you are not implementing this part now)_ which will do the chat UI and messages from the users will be saved as message files in the repository, then it will wait until the agent answers the message - writes an agent answer into the file and show it back to the user
-   The `ptbk agent` will look at folder `messages/queued` and list all the markdown files, in one run it will run the coding runner with prompt above and replace the PATH_TO_FILE in prompt with path to that markdown. When the runner finishes, the file should be moved to `messages/finished` folder, so the next run of `ptbk agent` will not answer the same message again, but it will look for the next unanswered message in `messages` folder and answer it, and so on, so you can add messages one by one into `messages/queued` folder and the agent will answer them one by one and move them to `messages/finished` folder
-   Reuse the same runners for `ptbk coder` and `ptbk agent`
-   In `ptbk agent` you are not working with emoji tags
-   The commit message should be "Answering message ..."
-   The `ptbk agent run` is running `ptbk agent tick` in the watch loop, when file with new message appears it runs `ptbk agent tick` and then waits for the next message, it runs one message at a time, it does not end after last message, but it keeps running and waiting for new messages, so you can add messages one by one and the agent will answer them one by one
-   The `ptbk agent tick` will anwer only one message _(or none if there are no unanswered messages)_ and then end
-   Keep in mind the DRY _(don't repeat yourself)_ principle, share as much code as possible between the `ptbk coder` and `ptbk agent` implementations
-   Do a proper analysis of the current functionality of `ptbk coder` and related functionality before you start implementing.
