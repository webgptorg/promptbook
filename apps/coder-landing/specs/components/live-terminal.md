# Component: Live terminal demo

A fake terminal in the [hero](../sections/hero.md) which replays a realistic `ptbk coder run` dashboard without running anything in the visitor's browser.

## Frame

Same window chrome as the [terminal block](./terminal-block.md) (rounded frame, `#0d1117` body, `#161b22` title bar, traffic lights), with two differences:

-   Title: `ptbk coder run - live terminal`.
-   Instead of a Copy button, a live indicator: a small pulsing green dot + the word `live` in Promptbook Green.
-   Body height is fixed and scrolls when the terminal is narrower or shorter than the full dashboard.

## Playback

1.  The session starts with `LIVE_RUN_COMMAND` from [`../content/commands.md`](../content/commands.md), typed character by character.
2.  The terminal then shows real dashboard snapshots generated by the shared `ptbk coder run` renderer in `scripts/run-codex-prompts/ui/buildCoderRunUiFrame.ts`.
3.  The top agent visual is resolved through the same `buildCoderRunAgentVisual` path as the CLI, which renders the agent's shared avatar visual (`META VISUAL AsciiOctopus`) through the shared avatar-to-ASCII pipeline used by Agents Server avatar visuals.
4.  Frames progress through loading, running, verifying, and done states. The final state mirrors a limited run ending with `Run limit reached after 1 prompt run.`
5.  When the frame sequence finishes, the terminal pauses briefly, clears, and replays from the typed command.

The story the sample must always tell: a real rich `ptbk coder run` session with Session, Current task, Live output, and Controls boxes; not a separate prose log of imagined agent work.
