# Visual design

The page uses a dark, terminal-oriented visual style without relying on external screenshots.

## Palette

-   Background: near-black charcoal.
-   Surface: dark neutral panels.
-   Text: warm off-white.
-   Muted text: grey-green.
-   Accents: green, teal, amber, and red.

The palette must avoid becoming a single-hue blue, purple, beige, or brown theme.

## Layout

-   Use full-width page bands with constrained inner content.
-   Use cards only for repeated items, FAQ entries, terminal frames, and product mockups.
-   Avoid cards inside cards.
-   Keep all sections responsive down to mobile width.

## Product visual

The primary product visual is a terminal and local kanban board mockup that reveals the actual `ptbk coder` workflow:

-   CLI command execution.
-   Prompt queue loading.
-   Verification attempts.
-   Commit completion.
-   Browser board columns for ready, running, draft, and done prompts.

## Typography

-   Use system fonts.
-   Use monospace only for commands, prompt status markers, and terminal output.
-   Do not scale font size directly with viewport width.
