# Information architecture

The landing page is a single-page website with anchored sections.

## Header

The header contains:

-   Product mark and name: `ptbk coder`
-   Section links:
    -   Commands
    -   Workflow
    -   Server
    -   Harnesses

## Sections

1. Hero: states the product promise and shows terminal plus kanban preview.
2. Terminal walkthrough: shows commands from installation through advanced operation.
3. Workflow: explains the prompt lifecycle.
4. Features: lists practical capabilities grounded in the CLI.
5. Server mode: explains the local kanban control surface.
6. Harnesses: lists supported runner integrations.
7. FAQ: answers developer comparison questions.
8. Footer: shows product ownership and the install command.

## Navigation behavior

Header links scroll to in-page anchors. No route changes or client-side state are required.
